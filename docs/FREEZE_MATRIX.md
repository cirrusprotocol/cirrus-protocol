# Freeze Matrix

This document lists files and surfaces that carry hard-freeze or canonical-sync constraints. Changes to these surfaces require additional validation steps before merging.

---

## Hard-freeze surfaces

These files must not be modified without explicit cross-language validation. They encode protocol constants, canonical hash preimage layouts, or domain tags that are independently re-derived in both Rust and TypeScript. A divergence or layout drift can break parity checks, validator-backed integration tests, or verifier security assumptions without being obvious from a single-language unit test.

| File | Constraint | Validation required |
|---|---|---|
| `programs/shielded_pool_anchor/src/instructions/attestation.rs` | Ed25519 threshold attestation parser. Layout and offset arithmetic must stay in sync with the Solana `Ed25519SigVerify` precompile instruction format. | Re-run all validator-backed integration suites (`anchor run test_all`). Re-audit against the precompile spec. |
| `lib/crypto.ts` | TypeScript mirror of `crates/shielded-pool-interface/src/instruction.rs`. Intent hash and handshake hash preimage layout, field order, and domain tags must exactly match the Rust canonical implementation. | Regenerate `tests/fixtures/parity_vectors.json` and re-run `anchor run test_parity`. Both sides must agree byte-for-byte. |
| `tests/fixtures/parity_vectors.json` | Golden vectors generated by the Rust canonical implementation. Must not be edited manually. | Regenerate only when `instruction.rs` layout or domain tags change. Re-run `anchor run test_parity` after regeneration. |

## Canonical-sync surfaces

Changes to these surfaces require parity vector regeneration or cross-system coordination:

| File / surface | Notes |
|---|---|
| `crates/shielded-pool-interface/src/instruction.rs` | Rust canonical hash implementation. Any layout or tag change here must be mirrored in `lib/crypto.ts` and parity vectors must be regenerated. |
| `programs/shielded_pool_anchor/src/state.rs` | Anchor account structs and event definitions. Adding or reordering fields requires migration instructions for existing deployed accounts and IDL snapshot update. |
| `idl/shielded_pool_anchor.json` | Tracked IDL snapshot. Refresh with `anchor run update_idl` after any rebuild. |

## Files that may evolve

These files are under active development and are not frozen:

| File / surface | Status |
|---|---|
| `lib/zk_indexer/` — all local indexer modules | Active development. Tests must pass; no cross-language parity dependency. |
| `scripts/` — operator and indexer scripts | Active development. No freeze constraints. |
| `docs/` — operator and security documentation | Must stay consistent with implemented behavior. |

## Parity vector regeneration procedure

Run only when `crates/shielded-pool-interface/src/instruction.rs` layout or domain tags change:

```bash
cargo run -p shielded-pool-interface --example parity_vectors \
  > tests/fixtures/parity_vectors.json
anchor run test_parity
```

Expected: `21 passing`. Commit the updated fixture together with the layout change in the same PR.
